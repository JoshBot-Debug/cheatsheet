# The graphics pipeline

### 1. **VTX**: Vertex Data
  * The raw data stream fed into the pipeline from the CPU. This includes vertex positions, normals, texture coordinates (UVs), and colors stored in **Vertex Buffer Objects** (**VBOs**).

```cpp
std::vector<GLfloat> vertices = {0.5f,  0.5f, 0.0f, 0.5f, -0.5f, 0.0f, -0.5f, -0.5f, 0.0f, -0.5f,  0.5f, 0.0f};
std::vector<GLuint> indices = {  0, 1, 3, 1, 2, 3};

GLuint VAO, VBO, EBO;
glGenVertexArrays(1, &VAO);
glGenBuffers(1, &VBO);
glGenBuffers(1, &EBO);

// Start recording the VBO layout and EBO binding
glBindVertexArray(VAO);

// Vertex Buffer
glBindBuffer(GL_ARRAY_BUFFER, VBO);
glBufferData(GL_ARRAY_BUFFER, vertices.size() * sizeof(GLfloat), vertices.data(), GL_STATIC_DRAW);

// Element Buffer
glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, EBO);
glBufferData(GL_ELEMENT_ARRAY_BUFFER, indices.size() * sizeof(GLuint), indices.data(), GL_STATIC_DRAW);

// OpenGL should read the vertex array attributes
// index -> location = 0
// size -> 3 components
// type-> float component
// normalized -> nope
// stride -> Byte offset between consecutive generic vertex attributes. In this case: x,y,z * sizeof(float)
// pointer-> Offset of the first component of the first generic vertex attribute in the array, in this case offset 0.
// https://registry.khronos.org/OpenGL-Refpages/gl4/html/glVertexAttribPointer.xhtml
// void glVertexAttribPointer(GLuint index, GLint size, GLenum type, GLboolean normalized, GLsizei stride, const void * pointer);
glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3 * sizeof(float), (void*)0);
// To use non-array data, call glDisableVertexAttribArray.
// To use array data, call glEnableVertexAttribArray. In this case, it's array data.
glEnableVertexAttribArray(0);

// Stop recording
glBindVertexArray(0);
glBindBuffer(GL_ARRAY_BUFFER, 0);

int render() {
  // ...
  glBindVertexArray(VAO);
  glDrawElements(GL_TRIANGLES, 6, GL_UNSIGNED_INT, 0);
  glBindVertexArray(0);
  // ...
}
```

### 2. **VS**: Vertex Shader
  * Executes once for each vertex.
  * Its primary job is to transform vertices from local object space into Clip Space using your Matrix transformations (MVP). It can also pass data (like texture coordinates) down the line.

```glsl
#version 450 core
layout (location = 0) in vec3 aPos; // Input from VTX
uniform mat4 u_MVP;

void main() {
  gl_Position = u_MVP * vec4(aPos, 1.0); 
}
```

### 3. **TCS**: Tessellation Control Shader (Optional)
  * The first part of the tessellation process. It determines how much to subdivide a patch of geometry by setting tessellation levels (`gl_TessLevelOuter` and `gl_TessLevelInner`).

```glsl
#version 450 core
layout (vertices = 3) out; // Output 3 vertices per patch

void main() {
  // Pass coordinate data through
  gl_out[gl_InvocationID].gl_Position = gl_in[gl_InvocationID].gl_Position;
  
  // Set tessellation levels (how fine the split is)
  if (gl_InvocationID == 0) {
    gl_TessLevelInner[0] = 4.0; // Subdivisions inside
    gl_TessLevelOuter[0] = 4.0; // Subdivisions on edges
    gl_TessLevelOuter[1] = 4.0;
    gl_TessLevelOuter[2] = 4.0;
  }
}
```

### 4. **TES**: Tessellation Evaluation Shader
* Calculates the actual positions of the new vertices generated by the tessellation engine using barycentric coordinates (`gl_TessCoord`).

```glsl
#version 450 core
layout (triangles, equal_spacing, ccw) in;

void main() {
  // Interpolate the position of the new vertex across the triangle patch
  //   glPatchParameteri(GL_PATCH_VERTICES, 3);
  // therefore, gl_in has 3 vertices
  vec3 p0 = gl_in[0].gl_Position.xyz;
  vec3 p1 = gl_in[1].gl_Position.xyz;
  vec3 p2 = gl_in[2].gl_Position.xyz;
  
  // gl_TessCoord holds barycentric coordinates (u, v, w)
  vec3 finalPos = gl_TessCoord.x * p0 + gl_TessCoord.y * p1 + gl_TessCoord.z * p2;
  
  gl_Position = vec4(finalPos, 1.0);
}
```
To render patches via C++, change your draw call from `GL_TRIANGLES` to `GL_PATCHES`:

```cpp
void render() {
  // ...
  glPatchParameteri(GL_PATCH_VERTICES, 3);
  glDrawElements(GL_PATCHES, 6, GL_UNSIGNED_INT, 0);
  // ...
}
```

::::note
`glPatchParameteri(GL_PATCH_VERTICES, 3);` `GL_PATCH_VERTICES` is set to `3`, so in the **TES**, `gl_in` will have 3 vertices/elements
::::

::::info
**Barycentric coordinates** are a system that specifies the location of a point inside or around a geometric shape (a simplex, such as a triangle or tetrahedron) using relative weights or masses assigned to the shape's vertices
::::

### 5. **GS**: Geometry Shader

* Takes full primitives and can generate new vertices or primitives using `EmitVertex()` and `EndPrimitive()`.

```glsl
#version 450 core
layout (triangles) in;
layout (triangle_strip, max_vertices = 3) out;

void main() {
  // Example: Simply pass the original triangle straight through
  for(int i = 0; i < 3; i++) {
    gl_Position = gl_in[i].gl_Position;
    EmitVertex(); // Construct a vertex
  }
  EndPrimitive();
}
```

### 6. **Rasterizer**

```cpp
glViewport(0, 0, 800, 600);          // mapping area to screen space
glCullFace(GL_BACK);                 // Cull back-facing triangles
glEnable(GL_CULL_FACE);              // Enable face culling 
glPolygonMode(GL_FRONT_AND_BACK, GL_FILL); // GL_FILL / GL_LINE / GL_POINT
```

### 7. **FS**: Fragment Shader
* Determines color of the pixel
* Runs once per pixel.

```glsl
#version 450 core
out vec4 FragColor;

void main() {
  FragColor = vec4(1.0f, 0.0f, 0.0f, 1.0f);  // A nice bright red screen :P
}
```

### 8. **FB**: Framebuffer / Per-Sample Operations

* Like the Rasterizer, this is a fixed-function blending and testing ground handled via API commands.

```cpp
glEnable(GL_DEPTH_TEST);  // Discard fragments hidden behind closer objects
glDepthFunc(GL_LESS);

glEnable(GL_BLEND);       // Alpha maps for transparency
glBlendFunc(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA);

// Clear buffers at start of loop
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT | GL_STENCIL_BUFFER_BIT);
```

### 9. **CS**: Compute Shader
* Bind textures, write to them, paint a picture, do math, etc. You can use shaders to do any computation.
* **The Compute Shader sits completely outside the traditional graphics rendering pipeline**
```glsl
#version 450 core
layout (local_size_x = 16, local_size_y = 16, local_size_z = 1) in;

layout(rgba32f, binding = 0) writeonly uniform image2D imgOutput;

void main() {
  ivec2 pixelCoords = ivec2(gl_GlobalInvocationID.xy);
  vec4 pixelColor = vec4(1.0, 0.0, 0.0, 1.0);
  
  imageStore(imgOutput, pixelCoords, pixelColor);
}
```
