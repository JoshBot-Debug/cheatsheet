# Depth Testing

Depth testing (the Depth Buffer) is used to prevent triangles rendering in the front while they're supposed to be behind other triangles

## What is the Depth Buffer?

The depth buffer is a non-visible 2D buffer that matches the width and height of your screen window. Instead of storing color data like the color buffer, each pixel in the depth buffer stores a depth value (*usually a float between `0.0` and `1.0`*).
* `0.0` represents the near plane.
* `1.0` represents the far plane.

**Depth testing is done in screen space after the fragment shader has run** (and after the stencil test).

The screen space coordinates relate directly to the viewport defined by OpenGL's `glViewport` function and can be accessed via GLSL's built-in `gl_FragCoord` variable in the fragment shader. 
The `x` and `y` components of `gl_FragCoord` represent the fragment's screen-space coordinates (with (0,0) being the bottom-left corner). 
The `gl_FragCoord` variable also contains a z-component which contains the depth value of the fragment. This `z` value is the value that is compared to the depth buffer's content.

::::note
The depth buffer's values are non-linear.
::::

::::note
Today most GPUs support a hardware feature called **early depth testing**. 

**Early depth testing is done after the rasterization stage, before the fragment shader**.
Whenever it is clear a fragment isn't going to be visible (it is behind other objects) we can prematurely discard the fragment.

A restriction on the fragment shader for early depth testing is that you shouldn't write to the fragment's depth value. 
If a fragment shader would write to its depth value, early depth testing is impossible;
OpenGL won't be able to figure out the depth value beforehand.
::::

## Enable depth testing

### When you don't need the depth texture
```cpp
glEnable(GL_DEPTH_TEST);  
```
The depth buffer is automatically created by the windowing system and stores its depth values as 16, 24 or 32 bit floats. In most systems you'll see a depth buffer with a precision of 24 bits.

Once enabled, OpenGL automatically stores fragments their z-values in the depth buffer if they passed the depth test and discards fragments if they failed the depth test accordingly. 
If you have depth testing enabled you should also clear the depth buffer before each frame using GL_DEPTH_BUFFER_BIT; otherwise you're stuck with the depth values from last frame:

```cpp
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
```

There are certain scenarios imaginable where you want to perform the depth test on all fragments and discard them accordingly, but not update the depth buffer. 
Basically, you're (temporarily) using a read-only depth buffer. OpenGL allows us to disable writing to the depth buffer by setting its depth mask to `GL_FALSE`:

```cpp
glDepthMask(GL_FALSE);
```

### When you need the depth texture

To use the depth buffer later—typically for post-processing effects like Depth of Field, screen-space shadows, or fog—you cannot easily read from the default screen framebuffer. 
Instead, you need to render your scene into a custom Framebuffer Object (**FBO**) and attach a texture specifically to hold the depth data.

Once rendered, that depth texture can be passed into a subsequent shader just like any normal color texture.

Generate the depth texture:
```cpp
unsigned int depthTexture;
glGenTextures(1, &depthTexture);
glBindTexture(GL_TEXTURE_2D, depthTexture);

// you must create a texture with a depth-specific format, such as GL_DEPTH_COMPONENT24 or GL_DEPTH_COMPONENT32F
glTexImage2D(GL_TEXTURE_2D, 0, GL_DEPTH_COMPONENT24, 800, 600, 0, GL_DEPTH_COMPONENT, GL_FLOAT, NULL);

glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_NEAREST);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_NEAREST);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
```

Attach the Texture to a Custom Framebuffer:
```cpp
unsigned int fbo;
glGenFramebuffers(1, &fbo);
glBindFramebuffer(GL_FRAMEBUFFER, fbo);

// Attach the depth texture to the FBO
glFramebufferTexture2D(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_TEXTURE_2D, depthTexture, 0);

// If you ONLY want depth and no color data in this FBO:
// glDrawBuffer(GL_NONE);
// glReadBuffer(GL_NONE);

if (glCheckFramebufferStatus(GL_FRAMEBUFFER) != GL_FRAMEBUFFER_COMPLETE)
    std::cout << "ERROR::FRAMEBUFFER:: Framebuffer is not complete!" << std::endl;

glBindFramebuffer(GL_FRAMEBUFFER, 0);
```

Render the Scene to the FBO:
```cpp
glBindFramebuffer(GL_FRAMEBUFFER, fbo);
glEnable(GL_DEPTH_TEST); 
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);

RenderScene();

glBindFramebuffer(GL_FRAMEBUFFER, 0);
```
::::note
Notice that we still had to use `glEnable(GL_DEPTH_TEST)`, `GL_DEPTH_BUFFER_BIT`. The only difference is we bound a custom **fbo** where we specified our custom depth attachment.
::::

Once you have written to the framebuffer, you're depth texture is ready to be bound to your shader program and read.


## Depth test function

OpenGL allows us to modify the comparison operators it uses for the depth test.

```cpp
glEnable(GL_DEPTH_TEST); // enable first
glDepthFunc(GL_LESS);
```

Function|Description
|:---|:---
GL_ALWAYS |	The depth test always passes.
GL_NEVER | The depth test never passes.
GL_LESS	| Passes if the fragment's depth value is less than the stored depth value.
GL_EQUAL | Passes if the fragment's depth value is equal to the stored depth value.
GL_LEQUAL	| Passes if the fragment's depth value is less than or equal to the stored depth value.
GL_GREATER	| Passes if the fragment's depth value is greater than the stored depth value.
GL_NOTEQUAL	| Passes if the fragment's depth value is not equal to the stored depth value.
GL_GEQUAL	| Passes if the fragment's depth value is greater than or equal to the stored depth value.

## Depth value precision

The depth value precision is non-linear, depth precision is highly dense near the near clipping plane and falls off exponentially as you approach the far clipping plane.

To view the depth buffer, try rendering this to see that depth is not linear.

```cpp
#version 330 core
out vec4 FragColor;

float near = 0.1; 
float far  = 100.0; 
  
float LinearizeDepth(float depth) 
{
    float z = depth * 2.0 - 1.0; // back to NDC 
    return (2.0 * near * far) / (far + near - z * (far - near));	
}

void main()
{             
    float depth = LinearizeDepth(gl_FragCoord.z) / far; // divide by far for demonstration
    FragColor = vec4(vec3(depth), 1.0);
    // FragColor = vec4(vec3(gl_FragCoord.z), 1.0);
}
```

## Z-fighting

A common visual artifact may occur when two planes or triangles are so closely aligned to each other that the depth buffer does not have enough precision to figure out which one of the two shapes is in front of the other.

**How to Fix Z-Fighting:**
* **Add a tiny offset**: Manually nudge one of the geometry pieces slightly forward or backward in 3D space.
* **Bring the near plane further away**: Setting your projection matrix near plane to `0.1` instead of `0.001` dramatically saves depth precision.
* **Use a higher bit depth buffer**: Upgrade from a 16-bit to a 24-bit or 32-bit depth buffer.
