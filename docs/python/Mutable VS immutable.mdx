## Mutable Vs Immutable objects in Python



* In Python every variable refrence to an objects which is an instance of a class.
* When an object is declared ,it is assigned to a unique object id.
* The type of object is declared at runtime and can't be changed afterward.However, for mutable objects- the value or content can change for mutable objects ***but never the type***.
1. ### Immutable objects
   * Once it is created , the value of the object can't be modified.
   * Immutable objects datatypes like ***int,str,float,bool,unicode and tuple***.
   * examples 
    ```py 
    tuple1=(0,2,4,6)
    tuple1[0]=7 # Throw TypeError: 'tuple' object doesn't support item assigment.

    message="Hello Python"
    message[1]=A #Throw TypeError: 'str' object doesn't support item
    ```
2. ### Mutable objects
   * Mutable objects datatypes like ***list,dict,set***.
   * Custom classes are mutable - When you create a custom class, python automatically gives for every instances , a private , hidden dictionary called ```__dict__```.This dictionary stores all attributes(variables) belonging to that specific object.Bcz of dictionary is mutable , we can add, delete or change the attributes inside the dictionary at anytime during runtime.
   * ***List***
       
       * It is mutable , it allows to change the value once it is created.
       * It provide methods like append(),insert(),extend(),remove() and pop().
       * Operations
        ```py
        my_list = [1, 2, 3]
        my_list.append(4)
        print(my_list) 

        my_list.insert(1, 5)
        print(my_list) 

        my_list.remove(2)
        print(my_list) 

        popped_element = my_list.pop(0)
        print(my_list)         
        print(popped_element)

        ```

        ****output****
        ```py
        [1, 2, 3, 4]
        [1, 5, 2, 3, 4]
        [1, 5, 3, 4]
        [5, 3, 4]
        1

        ```
    * **Dictionary**
         
        ```py
        my_dict = {"name": "Ram", "age": 25}
        new_dict = my_dict
        new_dict["age"] = 37

        print(my_dict)   
        print(new_dict)
        ```
        ***output***

        ```
        {'name': 'Ram', 'age': 37}
        {'name': 'Ram', 'age': 37}
        ```

    * **Set**
        ```
        my_set = {1, 2, 3}
        new_set = my_set
        new_set.add(4)

        print(my_set)    
        print(new_set)
        ```

        ***output***

        ```
        {1, 2, 3, 4}
        {1, 2, 3, 4}
        ```
    * **MUtable VS Immutable**
        
        * Mutable objects are quicker to access and expensive to change bcz of it involves the creation of the copy.Whereas mutable objects are easy to change
        * Mutable object recommended when there is a change of size or contect of the object.
        * ***Exception*** - The immutable objects can contains mutable objects.
  
3. ### Under the Hood : ```id()``` and ```hash()```
   * ```id()``` **Function**
  
        * ```id()``` function returns the **actual memory address** of an object.
        * If ```id()``` stays the same after an operation, the object is mutable.
        * if ```id()``` changes, a new object is created(immutable behavior)
  
        ```py
        # Immutable Example (String)
        name = "Alice"
        print(id(name))  # Output: 140123456 (example address)

        name += " Smith" 
        print(id(name))  # Output: 140987654 (The address CHANGED! A new string was made)

        # Mutable Example (List)
        numbers = [1, 2, 3]
        print(id(numbers))  # Output: 140555555

        numbers.append(4)
        print(id(numbers))  # Output: 140555555 (The address is EXACTLY the same)
        ```

    * ```hash()``` function

        * A hash value is fixed size integer that represents the contents of an object.
        * **Only immutable objects can be hashed**.Because their hash value remains constant.
        * **Mutable objects can't be hashed**. if the value of the object can change, its hash would change too.
  
        ```py
        print(hash("Alice"))     # Works fine (Immutable)
        print(hash((1, 2, 3)))   # Works fine (Tuple is immutable)

        # print(hash([1, 2, 3])) # Throws TypeError: unhashable type: 'list'
        ```

4. The ```copy``` Module: Shallow vs Deep copy

* When we assign a variable to another(eg : ```list_b=list_a```),python doesn't copy the data. It just creates a pointer to the ***same exact object*** in memory. if you change ```list_b``` , ```list_a``` also changes.
* To actually dublicate the data , use ```copy``` module - which features **shallow copy** and **deep copy**.
* The distinction will be critical for nested structures.
* #### Shallow copy (```copy.copy()```)
  * A shallow copy creates a new outer object, but ***popultes it with references to the exact same child objects*** as the original.
  * if we change the non nested element in the copy, the original is uneffected.
  * if we change the nested element in the copy the original will be affected.
* ### Deep copy (```copy.deepcopy()```)
   * A deep copy is completely independent.It creates a new outer object and ***recursively copies all child objects*** found inside the original.Changes made anywhere in the deep copy not affected in the original.

    ```py 
    import copy

    original=[[1,3,4,5],[2,5,6,8]]

    shallow_copy=copy.copy(original)

    deep_copy=copy.deep_copy(original)

    # The test , modufy a nested element in the shallow copy

    shallow_copy[0][0]=99
    
    print("Original:",original)  # [[99,3,4,5],[2,5,6,8]] -> CHANGED!
    print("Shallow:",shallow_copy) # [[99,3,4,5],[2,5,6,8]] 
    print("Deep:",deep_copy) #[[1,3,4,5],[2,5,6,8]] -> SAFE!


    ```
    
    #### Rule
    Use ```copy.copy()``` if your datastructure is entirely flat. Use ```copy.deepcopy()``` if you have nested mutable objects and want to ensure 100% independence between the original and the copy.
