## 1- Generators

* Generator is special type of function that returns an iterator object.
* Insted of using return keyword to send single value, generator function uses yeild to produce series of values overtime.
* The Generator function **pauses its execution** after yield, **maintaining (retaining or freezes) its state** between iterations.
* Each call to **next()** on generator object , resumes its execution right afetr the yield statement.
* Example
  ```py
  def func(max):
    cnt=1
    while cnt <=max:
        yield cnt
        cnt+=1

  ctr=func(5)
  for n in ctr:
    print(n)

  ```
  **output**
  ```
  1
  2
  3
  4
  5
  ```

## 2 - Need of Generators

### Memory efficeint 
Handle large or infinite data without loading everything into memory.

### NO list overhead
Yield items one by one at a time ,avoiding full list creation.

### Lazy Evaluation
Lazy evaluation measns "Dont do the work, dont use the memory until the code absolutely asks for it".
So only while calling next() , then next iteration happens.

### Support infinfte sequences
Ideal for generating unbounded data like Fibonacci series.

### Pipeline processing
Chaining generators (built by passing one generator into another.) , to process data in stages efficeintly.

* **The scinario cleaning a list of text**
  * cleain up extra spaces
  * Capitalize the names
  * Keep only the names start with the letter 'A'

    Here is how the generator chains together.

    ```py
    # Our raw data source
    raw_names = ["  alice ", "  bob  ", " charlie ", " alexandRA  "]

    # Stage 1: Strips white spaces
    strip_stage = (name.strip() for name in raw_names)

    # Stage 2: Capitalizes the words
    caps_stage = (name.capitalize() for name in strip_stage)

    # Stage 3: Filters only 'A' names
    final_pipeline = (name for name in caps_stage if name.startswith("A"))

    # --- AT THIS POINT: Zero work has been done. ---
    # The pipeline is built, but nobody has asked for data yet.

    print("Starting the loop...")
    for name in final_pipeline:
        print(f"Result: {name}")

    ```
    The data moves as "Pull effect". When the ```for name in fina_pipeline``` loop runs, it pulls from the end of the line.Here is the exact chain reaction.

    ```
    [Loop] says: "Hey Final Stage, give me a name!"
    ↓
    [Final Stage (Filter 'A')] says: "I don't have one yet. Hey Caps Stage, give me a capitalized name!"
    ↓
    [Caps Stage] says: "I don't have one either. Hey Strip Stage, strip something for me!"
    ↓
    [Strip Stage] grabs "  alice " from the raw list, strips it to "alice", and hands it to Caps Stage.
    ↓
    [Caps Stage] takes "alice", turns it into "Alice", and hands it to the Final Stage.
    ↓
    [Final Stage] checks "Alice". It starts with 'A'! It hands it to the Loop.
    ↓
    [Loop] prints: "Result: Alice"
    ```
    #### Why this is highly efficeint

    If you did this with regular python list, Python would:
      * Loop through all names to strip and save a new list to memory.
      *  Loop through all stripped names to capitalizes them and save another new list to memory
      *  Loop through all the capitalized names to filter them and save a third list to memory.
   * With **Generator Pipeline**, there are no intermediate lists are created in computer's RAM.It is a single continues stream where one item goes all the way through the pipeline before next item even enters.

## 3- Yield Vs Return

* ### Yeild :
   It is used in generator function to provide a sequence of values over time.When yield is executed it pauses the function, returns the current value and **retains the state of the function**.This allows the functions to continues from same point when called again through **next()**, making it ideal for generating large or complex sequences efficeinlty.

* ### Return:
  It is used to exit from the function and return a final value.Once return is executed , function is terminated immediately and no state is retained.This is suitabel for cases where single value is neede from a function.

## 4- Shorthand : Generator expression

* Generator expression are concise(short) way to create generators.
* They are similar to list comprehensation but use parentheses instead of square brackets and more memory efficeint.
* Example 
  ```py
  sq=(x*x for x in range(1,5))
  for in sq:
    print(i)

  ```
  **output**
  ```

  4
  ```

## 5- Shortcut tools

1. ### ``` yield from ```
    ``` yield from ``` is a shortcut used inside the generator to yield all values from another iterable (list,tuple, or another generrator) without using the ```for ``` loop.
    * old way 
    ```
    def combine_lists_old():
    list_a = [1, 2]
    list_b = [3, 4]
    
    # We have to loop through list_a manually
    for num in list_a:
        yield num
        
    # We have to loop through list_b manually
    for num in list_b:
        yield num

    # Test it
    for x in combine_lists_old():
        print(x)  # Outputs: 1, 2, 3, 4
    ```
    * The cleaner way 
     ```
     def combine_lists_new():
    list_a = [1, 2]
    list_b = [3, 4]
    
    # Shortcut! "Yield everything inside list_a, then move on"
    yield from list_a
    yield from list_b

    # Test it
    for x in combine_lists_new():
        print(x)  # Outputs: 1, 2, 3, 4
    ```
    * WHY USEFULL
       * Saves line of code 
       * Perfect for flatteinig the data
         * If you have nested structures (like a list of lists) or chained generators in a pipeline, yield from lets you seamlessly pass data up the chain without nesting multiple for loops


## 6- Advantages of Generators

* Useful for procesing large files like logs, because generator handle it in smaller parts ,**only holds one logs in memory at any given second**, saving memory and we dont wants to load the entire file at once.
* The iterator can do similar tasks, Generators are quicker bcz of we dont wants define __next__ and __iter__ methods manually.
